### typedef RPMessage_lld_recvCallback
```
typedef void(* RPMessage_lld_recvCallback) (void *handle, RPMessage_EpObject *obj, void *arg, void *data, uint16 dataLen, sint32 crcStatus, uint16 remoteCoreId, uint16 remoteEndPt);
```


**Brief**: 
Callback that is invoked when a message is received from any CPU at the specified local end point. 

The callback can be optionally registered during [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)


**Parameters**: 

  
* **obj** [in] RPMessage end point object created with [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)
  
* **arg** [in] Arguments specified by user during [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)
  
* **data** [in] Pointer to message 
  
* **dataLen** [in] Length of message 
  
* **remoteCoreId** [in] Core ID of sender 
  
* **remoteEndPt** [in] End point of sender 


**Note**: All message contents MUST be consumed in the callback. When callback returns the message buffer is released back to the sender. If the message contents are needed for deferred processing then take a copy of the message contents



### typedef RPMessage_lld_recvNotifyCallback
```
typedef void(* RPMessage_lld_recvNotifyCallback) (void *handle, RPMessage_EpObject *obj, void *arg);
```


**Brief**: 
Callback that is invoked when a message is received from any CPU at the specified local end point. 

The callback can be optionally registered during [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)


**Parameters**: 

  
* **obj** [in] RPMessage end point object created with [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)
  
* **arg** [in] Arguments specified by user during [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)


**Note**: 

  * Unlike [RPMessage_lld_recvCallback](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#typedef-rpmessage_lld_recvcallback), this callback only notifies that there is one or more messages to be read, but the message itself is not read by the driver unless RPMessage_lld_recv is called in this callback or later within a task.
  * If [RPMessage_lld_recvCallback](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#typedef-rpmessage_lld_recvcallback) is set, then [RPMessage_lld_recvNotifyCallback](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#typedef-rpmessage_lld_recvnotifycallback) callback is not used.

### function RPMessage_lld_send

```
sint32 RPMessage_lld_send(
    RPMessageLLD_Handle hRpMsg,
    void * data,
    uint16 dataLen,
    uint16 remoteCoreId,
    uint16 remoteEndPt,
    uint16 localEndPt,
    uint32 timeout
)
```


**Brief**: 
Announce a local end point at which a `service` is created to a remote core. 


Send a message to a remote core at a specified remote end point


**Parameters**: 

  
* **remoteProcId** [in] The remote core to annouce to. 
  
* **localEndPt** [in] Local end point of the service that is being announced 
  
* **name** [in] Name of the service that is being announced
  
* **data** [in] Pointer to message data to send 
  
* **dataLen** [in] size of message data to send 
  
* **remoteCoreId** [in] Remote core ID to whom the message is being sent 
  
* **remoteEndPt** [in] Remote core end point ID to which the message is being sent 
  
* **localEndPt** [in] Local end point that is sending the message 
  
* **timeout** [in] Amount of time to wait, in units of system ticks


**Return**: 

  * MCAL_SystemP_SUCCESS, when the annouce message was sent, else failure
  * MCAL_SystemP_SUCCESS, when the send message was successful 
  * SystemP_TIMEOUT, message not sent since free transmit buffer not available and timeout happened. 


**Note**: 

  * Announcing end points is optional and is not used internally by IPC RPmessage in any way. 
  * User MUST announce one by one to all remote core's of interest. There is no announce to `ALL` option 
  * To handle announcement messages, make sure user handler is registered during [RPMessage_lld_init](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_init). 
  * It is upto the end user to use the callback to signal or wait until a remote service is announced.
  * `dataLen` MUST be <= initParams.vringMsgSize - 16 bytes for internal header 
  * In order for a remote core to receive the message, a end point should be created on the remote core at the same value as `remoteEndPt`
  * localEndPt, is strictly not needed, however this is available to the user on the remote core and can be used as a reply end point. Use RPMessage_getLocalEndPt to set to the RPMessage object at which to listen for replies 
  * When timeout is 0, then if a free buffer is not available to transmit, it will return will immediately SystemP_TIMEOUT. Else it will wait for specified `timeout` ticks for a free buffer to be available.


### function RPMessage_controlEndPtDeInit

```
void RPMessage_controlEndPtDeInit(
    RPMessageLLD_Handle hRpMsg
)
```


**Brief**: 
Blocking API to wait till a message is received from any CPU at the specified local end point. 

**Parameters**: 

  
* **obj** [in] RPMessage end point object created with [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)
  
* **data** [in] Pointer to received message contents 
  
* **dataLen** [in] Length of user message buffer, in bytes 

 [out] Size of received message, in bytes 
  
* **remoteCoreId** [out] Core ID of sender 
  
* **remoteEndPt** [out] End point of sender 
  
* **timeout** [in] Time in system ticks to block for a message receive


**Return**: 

  * MCAL_SystemP_SUCCESS, new message received, all output parameters are valid 
  * SystemP_TIMEOUT, API unblocked due to timeout and output parameters should not be used. 


**Note**: 

  * Local end point is specified during [RPMessage_lld_construct](../Modules/group___d_r_v___i_p_c___r_p_m_e_s_s_a_g_e___m_o_d_u_l_e.md#function-rpmessage_lld_construct)
  * When callback is registered this API should not be used. 
  * `dataLen` when passed by user contains the user message buffer size, i.e size of buffer pointer to by 'data`. If received message size exceeds *dataLen then it is truncated. If received message size is <= *dataLen then all received bytes are copied to`data`and *dataLen indicates the size of valid bytes in`data`
