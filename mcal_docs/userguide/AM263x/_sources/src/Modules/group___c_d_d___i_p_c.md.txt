### enum Cdd_IpcStatusType
| Enumerator | Value | Description |
| ---------- | ----- | ----------- |
| CDD_IPC_UNINIT | 0U| The Complex Driver is not initialized or not usable.   |
| CDD_IPC_IDLE | 1U| The Complex Driver is not currently writing.   |
| CDD_IPC_ALLOW_ACCESS | 2U| The Complex Driver is currently open.   |
| CDD_IPC_BUSY | 3U| The Complex Driver is performing a Write.   |
| CDD_IPC_CLOSE | 4U| The Complex Driver is currently open.   |




**Brief**: 
This type defines a range of specific status for Complex driver. 

Requirements : AUTORADAR_MCAL-70

### function IpcNotifyUtils_lld_syncAll

```
sint32 IpcNotifyUtils_lld_syncAll(
    IpcNotifyUtils_Handle hIpcNotifyUtils,
    uint32 timeout
)
```


**Brief**: 
Send a message to all enabled cores and wait for sync message from all enabled cores. 

This API when called on all CPUs, make sure all CPUs execute upto a certain and then proceed only when all other CPUs have also executed to the same point.

This is useful esp during system init to make sure message exchange can be started only after all CPUs have finished their system initialization.


**Parameters**: 

  
* **timeout** - Amount of time in units of ticks to wait for the sync 


**Returns**: 

  
* **MCAL_SystemP_SUCCESSall** sync messages recevied successfully 
<br> 
  
* **SystemP_TIMEOUTsome** sync messages was NOT recevied after `timeout` ticks 
<br> 
  
* **SystemP_FAILUREinvalid** arguments 
<br> 


**Return**: sint32
