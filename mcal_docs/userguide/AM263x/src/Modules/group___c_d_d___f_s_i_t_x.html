5.5.2.3. function Cdd_FsiTx_Ping

Std_ReturnType Cdd_FsiTx_Ping(
    Cdd_FsiTx_HWUnitType HwUnitId
)

Brief: This service transmits the Ping frame to the receiver for establishing communication link.

The Ping frame with Tag 0 is transmitted initially. If the Tx is on the receiver hardware side, then once Tag 0 is received and gets acknowledged, Tx sends Tag1 Ping frame.

Sync/Async - Synchronous

Reentrancy - Reentrant

Parameters:

  • HwUnitId - Hardware Instance

Returns:

  • E_OK - Ping frame transmitted successfully

  • E_NOT_OK - Ping frame didn’t transmit successfully

Return: Std_ReturnType

5.5.2.4. function Cdd_FsiTx_BufferLoad

Std_ReturnType Cdd_FsiTx_BufferLoad(
    Cdd_FsiTx_HWUnitType HwUnitId,
    Cdd_FsiTx_DataBufferType * SrcBufferPtr,
    Cdd_FsiTx_DataType Cdd_FsiTx_userData,
    Cdd_FsiTx_BufferLengthType TxDataLength
)

Brief: Copy the data to be transmitted through Tx to the Tx buffer.

Sync/Async - Synchronous

Reentrancy - Reentrant

Parameters:

  • HwUnitId - The Tx HwUnit Instance which transmits data

  • SrcBufferPtr - Source Buffer from data to be copied.

  • Cdd_FsiTx_userData - User Data sent from application for each frame.

  • TxDataLength - Number of words to be copied for transmission.

Returns:

  • Returns the bufferload status

Return: Std_ReturnType

5.5.2.7. function CddFsiTx_GetStatus

Cdd_FsiTx_StatusType CddFsiTx_GetStatus(
    void 
)

Brief: This service returns the current state information of the driver.

Sync/Async - Synchronous

Reentrancy - Reentrant

Returns:

  • CDD_FSI_TX_UNINIT - Tx module has not initialized yet

  • CDD_FSI_TX_INIT - Tx module has initialized

  • CDD_FSI_TX_IDLE - Transmission through the Tx channel hasn’t started

  • CDD_FSI_TX_BUSY - Transmission has been started and is still going on

Return: Cdd_FsiTx_StatusType
