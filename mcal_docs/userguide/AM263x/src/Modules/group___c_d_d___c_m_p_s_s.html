5.3.3.4. function Cdd_Cmpss_ConfigHighComparator

void Cdd_Cmpss_ConfigHighComparator(
    Cdd_Cmpss_HwUnitType HwUnitId,
    uint16 Config
)

Brief: Sets the configuration for the high comparator.

This function configures a comparator. The Config parameter is the result of a logical OR operation between a CDD_CMPSS_INSRC_xxx value and if desired, CDD_CMPSS_INV_INVERTED and CDD_CMPSS_OR_ASYNC_OUT_W_FILT values.

The CDD_CMPSS_INSRC_xxx term can take on the following values to specify the high comparator negative input source:

  • CDD_CMPSS_INSRC_DAC - The internal DAC.

  • CDD_CMPSS_INSRC_PIN - External ‘INL’ pin. This is not applicable for CMPSSBx

CDD_CMPSS_INV_INVERTED may be ORed into Config if the comparator output should be inverted.

CDD_CMPSS_OR_ASYNC_OUT_W_FILT may be ORed into Config if the asynchronous comparator output should be fed into an OR gate with the latched digital filter output before it is made available for CTRIPH or CTRIPOUTH.

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • Config is the configuration of the high comparator.

5.3.3.6. function Cdd_Cmpss_ConfigOutputsHigh

void Cdd_Cmpss_ConfigOutputsHigh(
    Cdd_Cmpss_HwUnitType HwUnitId,
    uint16 Config
)

Brief: Sets the output signal configuration for the high comparator.

This function configures a comparator’s output signals CTRIP and CTRIPOUT. The Config parameter is the result of a logical OR operation between the CDD_CMPSS_TRIPOUT_xxx and CDD_CMPSS_TRIP_xxx values.

The CDD_CMPSS_TRIPOUT_xxx term can take on the following values to specify which signal drives CTRIPOUTH:

  • CDD_CMPSS_TRIPOUT_ASYNC_COMP - The asynchronous comparator output.

  • CDD_CMPSS_TRIPOUT_SYNC_COMP - The synchronous comparator output.

  • CDD_CMPSS_TRIPOUT_FILTER - The output of the digital filter.

  • CDD_CMPSS_TRIPOUT_LATCH - The latched output of the digital filter.

The CDD_CMPSS_TRIP_xxx term can take on the following values to specify which signal drives CTRIPH:

  • CDD_CMPSS_TRIP_ASYNC_COMP - The asynchronous comparator output.

  • CDD_CMPSS_TRIP_SYNC_COMP - The synchronous comparator output.

  • CDD_CMPSS_TRIP_FILTER - The output of the digital filter.

  • CDD_CMPSS_TRIP_LATCH - The latched output of the digital filter.

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • Config is the configuration of the high comparator output signals.

5.3.3.8. function Cdd_Cmpss_GetStatus

uint16 Cdd_Cmpss_GetStatus(
    Cdd_Cmpss_HwUnitType HwUnitId
)

Brief: Gets the current comparator status.

This function returns the current status for the comparator, specifically the digital filter output and latched digital filter output.

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • CDD_CMPSS_STS_LO_LATCHFILTOUT - Latched value of low digital filter output

5.3.3.9. function Cdd_Cmpss_ConfigDAC

void Cdd_Cmpss_ConfigDAC(
    Cdd_Cmpss_HwUnitType HwUnitId,
    uint16 Config
)

Brief: Sets the configuration for the internal comparator DACs.

This function configures the comparator’s internal DAC. The Config parameter is the result of a logical OR operation between the CDD_CMPSS_DACVAL_xxx, CDD_CMPSS_DACREF_xxx, and CDD_CMPSS_DACSRC_xxx.

The CDD_CMPSS_DACVAL_xxx term can take on the following values to specify when the DAC value is loaded from its shadow register:

  • CDD_CMPSS_DACVAL_SYSCLK - Value register updated on system clock.

  • CDD_CMPSS_DACVAL_PWMSYNC - Value register updated on PWM sync.

The CDD_CMPSS_DACREF_xxx term can take on the following values to specify which voltage supply is used as reference for the DACs:

  • CDD_CMPSS_DACREF_VDDA - VDDA is the voltage reference for the DAC.

  • CDD_CMPSS_DACREF_VDAC - VDAC is the voltage reference for the DAC.

The CDD_CMPSS_DACSRC_xxx term can take on the following values to specify the DAC value source for the high comparator’s internal DAC:

  • CDD_CMPSS_DACSRC_SHDW - The user-programmed DACVALS register.

  • CDD_CMPSS_DACSRC_RAMP - The ramp generator RAMPSTS register

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • Config is the configuration of the internal DAC.

5.3.3.34. function Cdd_Cmpss_ConfigFilterHigh

void Cdd_Cmpss_ConfigFilterHigh(
    Cdd_Cmpss_HwUnitType HwUnitId,
    uint16 SamplePrescale,
    uint16 SampleWindow,
    uint16 Threshold
)

Brief: Configures the digital filter of the high comparator.

This function configures the operation of the digital filter of the high comparator.

The SamplePrescale parameter specifies the number of system clock cycles between samples. It is a 10-bit value so a number higher than 1023 should not be passed as this parameter. The prescale used by digital filter is 1 more than SamplePrescale value. So, the input provided should be 1 less than the expected prescale.

The SampleWindow parameter configures the size of the window of FIFO samples taken from the input that will be monitored to determine when to change the filter output. This sample window may be no larger than 32 samples.

The Threshold parameter configures the Threshold value to be used by the digital filter.

The filter output resolves to the majority value of the sample window where majority is defined by the value passed into the Threshold parameter. For proper operation, the value of Threshold must be greater than SampleWindow / 2.

To ensure proper operation of the filter, the following is the recommended function call sequence for initialization:

  1. Configure and enable the comparator using Cdd_Cmpss_ConfigHighComparator() and Cdd_Cmpss_EnableModule()

  2. Configure the digital filter using Cdd_Cmpss_ConfigFilterHigh()

  3. Initialize the sample values using Cdd_Cmpss_InitFilterHigh()

  4. Configure the module output signals CTRIP and CTRIPOUT using Cdd_Cmpss_ConfigOutputsHigh()

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • SamplePrescale is the number of system clock cycles between samples.

  • SampleWindow is the number of FIFO samples to monitor.

  • Threshold is the majority Threshold of samples to change state.

5.3.3.37. function Cdd_Cmpss_ConfigRamp

void Cdd_Cmpss_ConfigRamp(
    Cdd_Cmpss_HwUnitType HwUnitId,
    const Cdd_Cmpss_RampConfigType * RampConfigPtr
)

Brief: Configures the comparator subsystem’s ramp generator.

This function configures many of the main settings of the comparator subsystem’s ramp generator. The MaxRampVal parameter should be passed the ramp maximum reference value that will be loaded into the ramp generator. The DecrementVal parameter should be passed the decrement value that will be subtracted from the ramp generator on each system clock cycle. The DelayVal parameter should be passed the 13-bit number of system clock cycles the ramp generator should delay before beginning to decrement the ramp generator after a PWMSYNC signal is received.

These three values may be be set individually using the Cdd_Cmpss_SetMaxRampValue(), Cdd_Cmpss_SetRampDecValue(), and Cdd_Cmpss_SetRampDelayValue() APIs.

The number of the PWMSYNC signal to be used to reset the ramp generator should be specified by passing it into the PwmSyncSrc parameter. For instance, passing a 2 into PwmSyncSrc will select PWMSYNC2.

To indicate whether the ramp generator should reset with the value from the ramp max reference value shadow register or with the latched ramp max reference value, use the UseRampValShdw parameter. Passing it true will result in the latched value being bypassed. The ramp generator will be loaded right from the shadow register. A value of false will load the ramp generator from the latched value.

Parameters:

  • HwUnitId HW ID. This param should be less than CDD_CMPSS_HW_UNIT_MAX

  • RampConfigPtr Pointer to the ramp configuration parameter. This should not be NULL.
