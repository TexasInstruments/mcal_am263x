Introduction

This is the release notes for MCAL AM26xx 11.01.00.02 done on 14-Nov-2025. The MCAL package consists of MCAL Driver & Applications for AM26xx family of devices. The MCAL modules are compliant to AUTOSAR specification versioned 4.3.1.

Licensing

Refer to manifest file mcal_manifest.html and license.txt present in installation base path

Getting Started

The AM263x HTML /AM263Px HTML /AM261x HTML User Guide provides the documentation and references necessary to begin development on TI's platforms using AM263x/AM263Px/AM261x. This document details about supported driver, installation, dependencies, build instructions, steps to run example applications. This release package also includes module specific User Guides, test reports, configurator User Guide and memory footprint details.

MCAL-34713 I2C: Declaration of I2C ISR added in 11.0 is breaking the CAT2 implementation in EB Stack CDD I2C Major AM263x, AM263Px, AM261x Fixed the code to not conflict with EB macros

MCAL-34848 FLS DAC Read Implementation error FLS Major AM263Px, AM261x DAC mode starts read only with aligned address. Fixed code to remove this restriction
MCAL-34894 Remove SchM_Enter_Fls_FLS_EXCLUSIVE_AREA_0 for FLS process jobs FLS Major AM263Px, AM261x The FLS driver has process jobs which is exclusive protected by SchM_Enter_Fls_FLS_EXCLUSIVE_AREA_0.

MCAL-34882 Fls_indac mode state machine error in interrupt mode FLS Major AM263Px, AM261x The Fls_JobNotification call is now called in process jobs only in polling mode. In interrupt mode, this is taken care in ISR.
MCAL-32445 FLS: Incorrect configuration for 4S-4S-4S and 4S-4D-4D modes FLS Major AM261x Updated the configuration
MCAL-31669 LIN: Not able to transmit master response frame after slave response frame LIN Major AM263x, AM263Px, AM261x
MCAL-32651 LIN: Unable to wakeup using 0xF0 wakeup key LIN Major AM263x, AM263Px, AM261x Invalid LIN wakeup key used earlier
MCAL-32717 Port_init fails in usermode PORT Major AM263x, AM263Px, AM261x

Versioning

Each package version is composed of 4 period-delimited numbers - represented here by the letters M, m, p and b [MM.mm.pp.bb]. The table below provides a descriptive reference regarding package version numbering.

Digit Meaning Description
MM Major revision Incremented when the new version is substantially different from the previous. For example, a new module added or an existing modules algorithm significantly altered
mm Minor revision Incremented when the new version has changed but not in a major way. For example, some minor changes in the API or feature set or bug fixes. Reset to 00 for any change to MM revision
pp Patch revision Incremented for patch/bug fix releases from any Major or Minor releases. Reset to 00 for any change to MM or mm revisions
bb Build revision Incremented for each release delivery build. Reset to 01 for any change to MM, mm or pp revisions
